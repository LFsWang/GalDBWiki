# Song 編輯指南

## 一、收錄門檻 (Scope)

GalDB 聚焦於具備 **敘事價值** 與 **功能性** 的音樂。

### 收錄對象

- **具備功能標籤**：於遊戲中擔當 OP、ED、Insert (插曲) 的樂曲。
- **官方推廣**：於官方宣傳片 (PV) 使用的主題曲或印象曲 (Image Song)。
- **BGM 特別條款**：採嚴格限縮。純背景音樂不予收錄，除非其在遊戲中具備上述功能（例如：無歌詞的樂曲被用作 ED，如 ひぐらしのなく頃に 目明し編《You》）。

### 不收錄範圍

- 純系統音效 (SE)、環境音、轉場短音效。
- 未在遊戲本體或官方媒體中出現的未採用曲。
- 僅存在於 OST CD 但在 VN 內無對應使用情境且不具代表性的填充曲。

### 關聯未驗證 (`is_unverified`)

- **用途**：標示「此曲目與本作品的關聯資料來源可疑、尚未經過志願者實際驗證」的情況，最典型的來源為 VNDB dump 自動匯入的資料。
- **勾選時機**：
    - 資料僅源自 VNDB（或其他第三方資料庫）的紀錄，編輯者本人 **未實際於遊戲中確認** 曲目確實使用。
    - 疑似為同名動畫、其他作品的誤植（如 VNDB 將動畫 OP 錯標至某 galgame）。
- **取消時機**：志願者實際遊玩本作品、於遊戲中確認曲目使用後，請一併取消勾選。若確認為誤植，應 **移除整條關聯** 而非僅取消此標記。
- **呈現效果**：
    - Song 頁面頂部顯示橙色警告橫幅。
    - VN 頁面、Staff 頁面的曲目列表會將該歌名以灰黑色字體呈現，滑鼠懸停顯示提示。
- **注意**：此為站內品質標記，**請勿於一般編輯時任意勾選**。對自己剛填入的新曲目勾選「未驗證」並無意義，應直接填入可信的資料或暫不建立該關聯。

## 二、條目拆分與聚合 (Normalization)

為了確保各版本影音資料的精確性，採 **「一版本一 ID」** 策略。

### 1. 強制拆分規則

- **演唱者不同**：即便旋律、歌詞一致，只要演唱者（或其名義）不同，應視為獨立版本（Song ID）。
- **內容差異**：包含歌詞修改、加入獨白、或編曲有顯著區隔（如 Acoustic vs Rock）。

### 2. 聚合機制 (Parent ID)

衍生版本必須透過 `parent_id` 標記其母體，並依據以下規則決定誰是母體：

- **平局決議 (Tie-break)**：若多版本地位平等（如全角色個人版且無主推歌手、無合唱版），依 **官方網站角色介紹序位首位** 為 Parent。否則以主推歌手版本、合唱版為 Parent。
- **版本更換 (Inheritance)**：若作品重製更換歌手，採 **「新指向舊」**，以最早發行之原版為 Parent。

## 三、欄位填寫規則

### 1. 標題與語言 (Song Names)

- **語言定義**：指該標題出現在「哪個語言版本」的遊戲中（而非歌詞語言）。
- **主要語言 (Main)**：優先勾選作品發源地的原始語言（通常為日文）。
- **拼音 (Latin/Kana)**：非拉丁字母標題應提供拼音以優化搜尋。
- **非官方標題 (粉絲翻譯 / 暫譯)**：與 VN 標準一致 — 可登記粉絲翻譯版本使用的名稱，但 **不應勾選 `is_official`**。

### 別名 (Alias)

- 填寫社群常用的別稱、副標、通稱。
- 用途為 **加強搜尋索引**，不影響顯示。

### 2. 演唱者 (Artists)

- **現階段支援職能**：僅限 **主唱 (Vocal)** 與 **和聲 (Chorus)**。
- **暫不收錄**：作詞、作曲、編曲 資訊現階段請勿填入歌曲條目（此類資訊目前請前往 VN 頁面 的 Staff 列表查看）。
- **以角色身分演唱 (As Character)**：
    - 若歌手以角色名義演唱，或該曲為角色在劇情中演唱，必須關聯角色 CID。
    - 渲染格式：系統將自動呈現為 `角色名 (C.V. 歌手名)`。

### 3. 主打影片 (Featured Video)

- **選取順序**（由上至下，擇一填入）：
    1. **官方完整影片**（YouTube／Bilibili／Niconico 官方 MV、官方 PV 完整版）
        - **與 VN 條目優先順序不同**：VN 條目偏好 OP／遊戲畫面版本（凸顯作品本身）；Song 條目偏好 **專輯／單曲 MV 版本**（凸顯歌曲本身），例如獨立 MV、歌手官方頻道的上傳。
    2. **社群完整影片**（具有年代感、音質尚可的非官方轉載）
    3. **合法平台預覽**（如 Apple Music／Spotify 的官方嵌入小工具，僅具試聽功能）
    4. **若以上皆無**：留空「主打影片」，僅在「外部連結」或備註中標註資料（如 VGMDb 連結）。
- **存取限制規範**：
    - **允許**：需年齡驗證（登入）的合法影片；提供免費試聽或預覽的合法平台嵌入。
    - **禁止**：完全需付費或登入後才可取得任何內容（無試聽、無預覽）。
    - **禁止**：為了填寫資料而 **自行上傳** 的盜版／侵權影片。
- **必填標記**：
    - **官方來源 (`video_is_official`)**：影片由官方（製作公司、發行商、歌手本人官方頻道）發布時勾選；社群轉載不勾。
    - **含成人／限制級內容 (`video_is_nsfw`)**：影片含成人、限制級或暴力內容時勾選，系統將於播放前顯示警語與遮罩。
- **判準說明**：
    - 「自行上傳」＝ 編輯者自己剪輯／上傳的侵權內容（禁止）。
    - 引用已存在於平台、非編輯者發布的社群影片，不受此條限制。
    - 主打影片 **留空是可接受的結果**，不應為了「填滿欄位」而降低規則一致性。

### 4. 簡介與法律警語 (Description)

- **歌詞禁令**：嚴禁填入歌詞全文（受著作權法保護）。請改用外部連結連結至合法歌詞網站（如 UtaTen, Uta-Net）。
- **授權連結**：若為創作者本人，請在專用連結框提供授權證明。

## 四、核心關聯結構 (Relations)

### 1. 關聯 VN (VN Relations)

一首歌若有多重用途（如同時是 OP 也是 ED），必須分別建立多條關聯。

- **屬性 (Type)**：OP, ED, Insert, Theme, Image, BGM（需具備上述功能）。
    - **Theme vs Image 區分**：`Theme` 對應 **故事／作品整體**（主題曲），`Image` 對應 **特定角色**（角色印象曲）。
    - **Theme / Image 特例**：此兩類 **不要求實際出現在遊戲本體中**，但需具備 **官方關聯性**（如收錄於官方原聲帶、官方釋出之角色印象專輯、官方 PV 使用等）。
- **序號 (Sequence)**：
    - **遊戲中同類型（OP／ED 等）有多首**時，依 **遊戲中出現的順序** 編號（如 OP1, OP2, ED1, ED2）。
    - **僅有一首**時 **留空不編號**，若需區分請改用「專屬角色」或「路線」欄位標註。
- **路線 (Route) 標準寫法**：自由註記，建議以英文填寫、簡短 15 字元以內（soft limit）。常用術語固定為 `True End` 與 `Grand End`，定義如下：

    #### `True End`（真結局）

    - **定義**：單一角色或核心敘事線的「最終正史」結局。它修正或超越了 Good End，是該路線邏輯上的終點。
    - **判定基準**：若遊戲中存在 Normal／Good／True 三層結構，則指向 True。
    - **範例**：《Summer Pockets》各角色的 True Route。

    #### `Grand End`（總括結局）

    - **定義**：「全篇章結束後」才解鎖，或跨越所有角色路線後的「大結局」。通常用來回收全作核心伏筆，而非僅某角色的結局。
    - **判定基準**：具備 **唯一性** 與 **最終性**。若作品同時有各角色的 True End 與一個全篇結尾，則最後那個稱為 Grand End。
    - **範例**：《Clannad》After Story 結尾、《Little Busters!》Refrain 結尾。

    #### 屬性優先級規則

    - **結構層級法**：
        - 若歌曲為了 **收束全作品** 而存在（通常是最後一次 staff roll 播放）→ 標記 `Grand End`。
        - 若歌曲僅出現在 **特定角色的最終路線** → 標記 `True End`。
    - **避免重複輸入**：若 Grand End 本身就是該遊戲唯一的 True End，**優先使用 `Grand End`**，以強調其作為全作頂點的特殊身分。
- **專屬角色 (Character)**：
    - 標記該曲為「特定角色專屬曲」。
    - 一首歌用於多角色時應分別建立關聯列；若為全角色通用則不填。
- **註記 (Row Note)**：以英文填寫、簡短 15 字元以內（soft limit）。標註多版本差異（如：`PS4 only`、`All-age`）。

### 2. 外部連結 (External Links)

目前支援的站點（未來陸續擴充）：

- **資料庫 (DB)**：MusicBrainz (Work)、MusicBrainz (Recording)、VGMdb (Album)
- **影音平台 (Platform)**：YouTube、YouTube Music、Bilibili、Chobit、Apple Music (JP)
- **歌詞網站 (Lyrics)**：UtaTen、Uta-Net

系統會自動偵測站點類型並進行網址正規化。
